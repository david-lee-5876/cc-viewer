# Workflow

多数のサブエージェントを決定論的にオーケストレーションする — ファンアウト、パイプライン、ループ、検証 — スクリプトを実行します。単一のコンテキストには広すぎる、不確実すぎる、または大きすぎる作業向けです。

## 使用タイミング

- 大きなタスクを分解し、多数のエージェントにまたがって並列にカバーする
- 結果を採用する前に、独立したまたは敵対的な検証でクロスチェックする
- 単一のコンテキストでは保持できない規模に取り組む: 移行、監査、広範な複数ファイルの一括処理

## 仕組み

- バックグラウンドで実行され、完了時に通知されます。`/workflows` でライブの進捗を確認できます。
- スクリプトは `agent()`、`parallel()`、`pipeline()`、`phase()` でエージェントを調整します。
- `pipeline()` は各アイテムをバリアなしでステージにストリームします (デフォルト)。`parallel()` はすべての結果を待つバリアです。
- スキーマを指定すると、各 `agent()` は自由テキストではなく検証済みの構造化データを返します。

## 注意事項

- ユーザーがマルチエージェントオーケストレーションを明示的に選択した場合にのみ実行されます。多数のエージェントを起動し、かなりの token を消費する可能性があります。
- 同時実行数はワークフローごとに上限が設けられています。超過したエージェントはキューに入り、スロットが空き次第実行されます。
- 単一のサブエージェントには、代わりに `Agent` ツールを使用してください — Workflow は真のファンアウトに留めてください。

## 関連概念

- `Agent` ツールを基盤とし、決定論的な制御フローのもとで多数のエージェントを実行します。

# ToolSearch

「遅延ツール」の完全なスキーマ定義をオンデマンドで取得し、呼び出せるようにします。多くのツールが利用可能な場合、一部は事前にロードされず、`<system-reminder>` メッセージの中に名前だけが現れます。スキーマが取得されるまでは名前しか分からず、パラメータ定義が存在しないため、そのツールは呼び出せません。`ToolSearch` はクエリを受け取り、遅延ツールのリストと照合し、一致したツールの完全な JSONSchema 定義を `<functions>` ブロックの中で返します。ツールのスキーマが結果に現れると、プロンプトの先頭で定義されたツールと全く同じように呼び出せるようになります。

## 使用タイミング

- 遅延ツールが必要な場合 — その名前は `<system-reminder>` に現れますが、トップレベルのツールリストにはパラメータ定義がありません。
- オンデマンドでロードされる MCP サーバーのツール（例: Slack、Gmail、computer-use）を使用したい場合。
- ある機能に対応する正確なツール名が分からず、キーワードで一度に候補を浮かび上がらせたい場合。

ツールのスキーマがすでにコンテキストにある場合は、再度検索しないでください — そのまま呼び出してください。

## パラメータ

- `query` (string, required): 遅延ツールを特定するために使用するクエリ。3 つの形式がサポートされています:
  - `select:Read,Edit,Grep` — これらの正確な名前でツールを取得します。
  - `notebook jupyter` — キーワード検索で、最大 `max_results` 件の最適な一致を返します。
  - `+slack send` — ツール名に `slack` が含まれることを必須とし、残りの語で順位付けします。
- `max_results` (number, optional): 返す結果の最大数。デフォルトは 5。

## 例

### 例 1: 正確な名前で取得

```
ToolSearch(query="select:WebFetch,WebSearch", max_results=5)
```

### 例 2: キーワード検索

```
ToolSearch(query="notebook jupyter", max_results=5)
```

### 例 3: MCP ツールキット全体を一度にロード

MCP サーバーのすべてのツール（例: computer-use）を一括ロードする場合、一つずつ選択するのではなく単一のキーワード検索を使用してください — サーバー名は部分文字列として、そのサーバー配下のすべてのツールに一致します:

```
ToolSearch(query="computer-use", max_results=30)
```

## 注意事項

- 遅延ツールを呼び出す前に、まず `ToolSearch` でそのスキーマを取得する必要があります — パラメータ定義が欠けているため、直接呼び出すと失敗します。
- ツールキット全体（例: MCP サーバーのすべてのツール）を一括ロードする場合は、ラウンドトリップを減らすために、多数の `select:` 呼び出しよりも一つのキーワード検索を優先してください。
- スキーマを取得すれば、そのツールは通常のツールと全く同じように動作します。同じツールを再検索しないでください。
- 結果は `<functions>` ブロックとして返り、各ツールは単一の `<function>{...}</function>` 行になります — トップレベルのツールリストと同じエンコーディングです。

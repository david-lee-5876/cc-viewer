# Workflow

執行一個以確定性方式編排大量子代理的指令稿——扇出、管線、迴圈與校驗——適用於範圍過廣、不確定性過高或單一脈絡無法容納的工作。

## 何時使用

- 拆解大型任務，並由多個代理平行涵蓋
- 在採信結論前，以獨立或對抗式校驗交叉驗證各項發現
- 承擔單一脈絡無法承載的規模：遷移、稽核、跨多檔案的大範圍梳理

## 工作原理

- 在背景執行；完成時你會收到通知。可用 `/workflows` 查看即時進度。
- 指令稿透過 `agent()`、`parallel()`、`pipeline()`、`phase()` 協調各代理。
- `pipeline()` 讓每個項目無柵欄地流經各階段（預設）；`parallel()` 是一個柵欄，會等待所有結果。
- 搭配 schema 時，每次 `agent()` 回傳經校驗的結構化資料，而非自由文字。

## 注意事項

- 僅在使用者明確選擇多代理編排時才執行；它可能衍生大量代理並消耗可觀的 token。
- 每個 workflow 的並行數有上限；超出的代理會排隊，待有空位再執行。
- 若只需單一子代理，請改用 `Agent` 工具——Workflow 應保留給真正的扇出場景。

## 相關概念

- 建構於 `Agent` 工具之上，在確定性控制流程下執行大量代理。

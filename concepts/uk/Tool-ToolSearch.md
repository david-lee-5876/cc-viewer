# ToolSearch

Отримує на вимогу повні визначення схем «відкладених інструментів», щоб їх можна було викликати. Коли доступно багато інструментів, частина з них не завантажується заздалегідь — вони зʼявляються лише за іменем усередині повідомлень `<system-reminder>`. Доки його схема не отримана, відоме лише імʼя, а визначення параметрів немає, тому інструмент неможливо викликати. `ToolSearch` приймає запит, зіставляє його зі списком відкладених інструментів і повертає повні визначення JSONSchema знайдених інструментів усередині блоку `<functions>`. Щойно схема інструмента зʼявляється в результаті, його можна викликати рівно так само, як будь-який інструмент, визначений на початку промпта.

## Коли використовувати

- Вам потрібен відкладений інструмент — його імʼя зʼявляється в `<system-reminder>`, але в списку інструментів верхнього рівня для нього немає визначення параметрів.
- Ви хочете використати інструменти MCP-сервера (наприклад, Slack, Gmail, computer-use), які завантажуються на вимогу.
- Ви не впевнені в точному імені інструмента для потрібної можливості і хочете за один раз вивести кандидатів за ключовим словом.

Якщо схема інструмента вже є в контексті, не шукайте знову — просто викличте його.

## Параметри

- `query` (string, обовʼязковий): Запит для пошуку відкладених інструментів. Підтримуються три форми:
  - `select:Read,Edit,Grep` — отримати ці конкретні інструменти за іменем.
  - `notebook jupyter` — пошук за ключовими словами, що повертає до `max_results` найкращих збігів.
  - `+slack send` — вимагати, щоб `slack` був присутній в імені інструмента, а потім ранжувати за рештою термінів.
- `max_results` (number, необовʼязковий): Максимальна кількість результатів для повернення. За замовчуванням — 5.

## Приклади

### Приклад 1: Отримання за точним іменем

```
ToolSearch(query="select:WebFetch,WebSearch", max_results=5)
```

### Приклад 2: Пошук за ключовими словами

```
ToolSearch(query="notebook jupyter", max_results=5)
```

### Приклад 3: Завантаження цілого MCP-набору за один раз

Під час масового завантаження всіх інструментів MCP-сервера (наприклад, computer-use) використовуйте один пошук за ключовим словом замість вибору кожного окремо — імʼя сервера як підрядок збігається з кожним інструментом цього сервера:

```
ToolSearch(query="computer-use", max_results=30)
```

## Примітки

- Перед викликом відкладеного інструмента ви повинні спершу отримати його схему за допомогою `ToolSearch` — прямий виклик завершиться помилкою, бо визначення параметрів відсутнє.
- Під час масового завантаження цілого набору (наприклад, усіх інструментів MCP-сервера) віддавайте перевагу одному пошуку за ключовим словом перед багатьма викликами `select:`, щоб скоротити кількість звернень.
- Щойно схему отримано, інструмент поводиться рівно як будь-який звичайний інструмент; не шукайте той самий інструмент повторно.
- Результати повертаються як блок `<functions>`, кожен інструмент — окремим рядком `<function>{...}</function>` — у тому самому кодуванні, що й список інструментів верхнього рівня.

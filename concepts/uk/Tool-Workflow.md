# Workflow

Запускає скрипт, який детерміновано оркеструє багато підагентів — fan-out, конвеєри, цикли та перевірку — для роботи, надто широкої, надто невизначеної або надто великої для одного контексту.

## Коли використовувати

- Розкласти велике завдання й покрити його паралельно багатьма агентами
- Перехресно перевірити висновки незалежною чи змагальною верифікацією, перш ніж покладатися на них
- Узятися за масштаб, який не вміщає один контекст: міграції, аудити, широкі багатофайлові обходи

## Як це працює

- Виконується у фоні; ви отримуєте сповіщення про завершення. Слідкуйте за прогресом наживо за допомогою `/workflows`.
- Скрипт координує агентів за допомогою `agent()`, `parallel()`, `pipeline()` та `phase()`.
- `pipeline()` пропускає кожен елемент через етапи без барʼєра (за замовчуванням); `parallel()` — це барʼєр, що чекає на всі результати.
- За наявності schema кожен `agent()` повертає валідовані структуровані дані замість вільного тексту.

## Примітки

- Запускається лише тоді, коли користувач явно обирає мультиагентську оркестрацію; може породити багато агентів і спожити значну кількість token.
- Конкурентність обмежена на кожен workflow; надлишкові агенти стають у чергу та виконуються в міру звільнення місць.
- Для одного підагента використовуйте натомість інструмент `Agent` — залиште Workflow для справжнього fan-out.

## Повʼязані поняття

- Будується на основі інструмента `Agent`, запускаючи багато агентів під детермінованим керуванням потоком.

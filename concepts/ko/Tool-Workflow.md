# Workflow

여러 서브에이전트를 결정론적으로 오케스트레이션하는 — 팬아웃, 파이프라인, 루프, 검증 — 스크립트를 실행합니다. 단일 컨텍스트에 담기에는 너무 광범위하거나, 너무 불확실하거나, 너무 큰 작업을 위한 것입니다.

## 사용 시점

- 큰 작업을 분해하여 여러 에이전트에 걸쳐 병렬로 처리하기
- 결과를 채택하기 전에 독립적이거나 적대적인 검증으로 교차 확인하기
- 단일 컨텍스트가 담을 수 없는 규모를 다루기: 마이그레이션, 감사, 광범위한 다중 파일 일괄 작업

## 작동 원리

- 백그라운드에서 실행되며, 완료 시 알림을 받습니다. `/workflows`로 실시간 진행 상황을 확인하십시오.
- 스크립트는 `agent()`, `parallel()`, `pipeline()`, `phase()`로 에이전트를 조율합니다.
- `pipeline()`은 각 항목을 배리어 없이 단계들로 스트리밍합니다 (기본값). `parallel()`은 모든 결과를 기다리는 배리어입니다.
- 스키마를 지정하면 각 `agent()`는 자유 텍스트 대신 검증된 구조화 데이터를 반환합니다.

## 참고사항

- 사용자가 다중 에이전트 오케스트레이션을 명시적으로 선택한 경우에만 실행됩니다. 많은 에이전트를 생성하고 상당한 token을 소비할 수 있습니다.
- 동시성은 워크플로우별로 상한이 있습니다. 초과된 에이전트는 큐에 들어가며 슬롯이 비는 대로 실행됩니다.
- 단일 서브에이전트의 경우 대신 `Agent` 도구를 사용하십시오 — Workflow는 진정한 팬아웃을 위해 남겨두십시오.

# Workflow

运行一个以确定性方式编排大量子代理的脚本——扇出、流水线、循环与校验——适用于范围过广、不确定性过高或单个上下文无法容纳的工作。

## 何时使用

- 拆解大型任务，并由多个代理并行覆盖
- 在采信结论前，用独立或对抗式校验交叉验证各项发现
- 承担单个上下文无法承载的规模：迁移、审计、跨多文件的大范围梳理

## 工作原理

- 在后台运行；完成时你会收到通知。可用 `/workflows` 查看实时进度。
- 脚本通过 `agent()`、`parallel()`、`pipeline()`、`phase()` 协调各代理。
- `pipeline()` 让每个条目无栅栏地流经各阶段（默认）；`parallel()` 是一个栅栏，会等待所有结果。
- 配合 schema 时，每次 `agent()` 返回经校验的结构化数据，而非自由文本。

## 注意事项

- 仅在用户明确选择多代理编排时才运行；它可能派生大量代理并消耗可观的 token。
- 每个 workflow 的并发数有上限；超出的代理会排队，待有空位再运行。
- 若只需单个子代理，请改用 `Agent` 工具——Workflow 应保留给真正的扇出场景。

## 相关概念

- 构建于 `Agent` 工具之上，在确定性控制流下运行大量代理。

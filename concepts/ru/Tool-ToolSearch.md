# ToolSearch

Извлекает по запросу полные определения схем «отложенных инструментов», чтобы их можно было вызвать. Когда доступно много инструментов, часть из них не загружается заранее — они появляются только по имени внутри сообщений `<system-reminder>`. Пока его схема не извлечена, известно лишь имя, а определения параметров нет, поэтому инструмент нельзя вызвать. `ToolSearch` принимает запрос, сопоставляет его со списком отложенных инструментов и возвращает полные определения JSONSchema найденных инструментов внутри блока `<functions>`. Как только схема инструмента появляется в результате, его можно вызывать ровно так же, как любой инструмент, определённый в начале промпта.

## Когда использовать

- Вам нужен отложенный инструмент — его имя появляется в `<system-reminder>`, но в списке инструментов верхнего уровня для него нет определения параметров.
- Вы хотите использовать инструменты MCP-сервера (например, Slack, Gmail, computer-use), которые загружаются по запросу.
- Вы не уверены в точном имени инструмента для нужной возможности и хотите за один раз вывести кандидатов по ключевому слову.

Если схема инструмента уже есть в контексте, не ищите снова — просто вызовите его.

## Параметры

- `query` (string, обязательный): запрос для поиска отложенных инструментов. Поддерживаются три формы:
  - `select:Read,Edit,Grep` — извлечь эти конкретные инструменты по имени.
  - `notebook jupyter` — поиск по ключевым словам, возвращающий до `max_results` наиболее подходящих совпадений.
  - `+slack send` — требовать, чтобы `slack` присутствовал в имени инструмента, затем ранжировать по остальным терминам.
- `max_results` (number, необязательный): максимальное число возвращаемых результатов. По умолчанию 5.

## Примеры

### Пример 1: извлечение по точному имени

```
ToolSearch(query="select:WebFetch,WebSearch", max_results=5)
```

### Пример 2: поиск по ключевым словам

```
ToolSearch(query="notebook jupyter", max_results=5)
```

### Пример 3: загрузка всего MCP-набора за один раз

При массовой загрузке всех инструментов MCP-сервера (например, computer-use) используйте один поиск по ключевому слову вместо выбора каждого по отдельности — имя сервера как подстрока совпадает с каждым инструментом этого сервера:

```
ToolSearch(query="computer-use", max_results=30)
```

## Примечания

- Перед вызовом отложенного инструмента вы должны сначала извлечь его схему с помощью `ToolSearch` — прямой вызов завершится ошибкой, потому что определение параметров отсутствует.
- При массовой загрузке целого набора (например, всех инструментов MCP-сервера) предпочитайте один поиск по ключевому слову множеству вызовов `select:`, чтобы сократить число обращений.
- Как только схема извлечена, инструмент ведёт себя ровно как любой обычный инструмент; не ищите тот же инструмент повторно.
- Результаты возвращаются как блок `<functions>`, каждый инструмент — отдельной строкой `<function>{...}</function>` — в той же кодировке, что и список инструментов верхнего уровня.

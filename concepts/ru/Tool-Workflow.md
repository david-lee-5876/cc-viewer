# Workflow

Выполняет скрипт, который детерминированно оркеструет множество подагентов — fan-out, конвейеры, циклы и проверку — для работы, которая слишком широка, слишком неопределённа или слишком велика для одного контекста.

## Когда использовать

- Декомпозировать большую задачу и покрыть её параллельно множеством агентов
- Перепроверить выводы независимой или состязательной проверкой, прежде чем на них полагаться
- Взяться за масштаб, который не помещается в один контекст: миграции, аудиты, широкие проходы по множеству файлов

## Принцип работы

- Выполняется в фоне; вы получаете уведомление по завершении. Следите за ходом в реальном времени через `/workflows`.
- Скрипт координирует агентов с помощью `agent()`, `parallel()`, `pipeline()` и `phase()`.
- `pipeline()` пропускает каждый элемент через стадии без барьера (по умолчанию); `parallel()` — это барьер, ожидающий все результаты.
- Со схемой каждый `agent()` возвращает валидированные структурированные данные вместо свободного текста.

## Примечания

- Выполняется только когда пользователь явно соглашается на многоагентную оркестрацию; может породить множество агентов и потребить значительное число token.
- Параллелизм ограничен для каждого workflow; лишние агенты встают в очередь и запускаются по мере освобождения слотов.
- Для одного подагента используйте вместо этого инструмент `Agent` — оставьте Workflow для настоящего fan-out.

## Связанные концепции

- Основан на инструменте `Agent`, запуская множество агентов под детерминированным потоком управления.
